# 风力发电机故障诊断系统 - 用户操作手册

> 版本：v1.0.0 | 适用对象：系统管理员、运维工程师、数据分析师、普通用户

---

## 一、系统概述

风力发电机故障诊断系统是一套面向风电运维场景的管理平台，提供用户管理、角色权限管理、操作审计等核心功能。系统采用深色科技风界面设计，支持 1080p 至 4K 分辨率自适应。

### 1.1 访问地址

| 服务 | 地址 | 说明 |
|------|------|------|
| 管理后台 | http://localhost:8081 | 前端界面 |
| 后端 API | http://localhost:8080 | 接口服务 |

### 1.2 浏览器要求

- Chrome 90+（推荐）
- Firefox 88+
- Edge 90+
- Safari 14+

### 1.3 预设账号

| 角色 | 用户名 | 密码 | 权限范围 |
|------|--------|------|----------|
| 系统管理员 | admin | admin123 | 全部功能 |

## 二、注册与登录

### 2.1 用户注册

1. 打开系统首页，点击「注册」标签页
2. 填写以下信息：
   - **用户名**（必填）：唯一标识，不可重复
   - **真实姓名**（选填）
   - **邮箱**（必填）：用于接收验证邮件
   - **密码**（必填）：至少 6 位
3. 点击「注册」按钮
4. 注册成功后，页面显示绿色提示卡片，提示验证邮件已发送

**注册流程：**

```
注册 → 待验证邮箱(状态3) → 邮箱验证 → 待审核(状态2) → 管理员审核 → 启用(状态1)
```

### 2.2 邮箱验证

1. 注册后前往邮箱查收验证邮件（主题：【风力发电机故障诊断系统】邮箱验证）
2. 点击邮件中的「验证邮箱」按钮或复制链接到浏览器打开
3. 验证成功后，账户进入「待审核」状态
4. 如未收到邮件，可在注册成功提示中点击「未收到邮件？重新发送」

> 注意：验证链接有效期为 30 分钟，过期需重新发送。

### 2.3 用户登录

1. 在登录标签页输入用户名和密码
2. 点击「登录」按钮
3. 登录成功后自动跳转到数据仪表盘

**登录限制：**

- 连续输错密码 5 次，账户将被自动锁定，需联系管理员解锁
- 待审核、待验证邮箱、已禁用的账户无法登录

### 2.4 退出登录

点击右上角用户头像区域，在下拉菜单中选择「退出登录」。

## 三、数据仪表盘

路径：首页 `/dashboard`

- 系统运行状态监控

## 四、用户管理

路径：系统管理 → 用户管理 `/system/user`

权限要求：需要「用户管理」菜单权限

### 4.1 查询用户

页面顶部提供搜索栏：

| 筛选条件 | 说明 |
|----------|------|
| 用户名 | 模糊匹配 |
| 真实姓名 | 模糊匹配 |
| 状态 | 精确匹配：禁用/启用/待审核 |

点击「搜索」执行查询，点击「重置」清空条件。

> 注意：查询结果受数据范围权限限制。管理员可查看全部用户，其他角色根据数据范围配置只能查看对应范围的用户。

### 4.2 新增用户

1. 点击右上角「新增用户」按钮
2. 在弹窗中填写：
   - **用户名**（必填）
   - **密码**（必填，至少 6 位）
   - **真实姓名**（选填）
   - **邮箱**（选填，格式校验）
   - **手机号**（选填，格式校验）
3. 点击「确定」完成创建

### 4.3 编辑用户

点击用户行的「编辑」按钮，可修改真实姓名、邮箱、手机号。用户名和密码不可在此修改。

### 4.4 用户状态管理

| 操作 | 按钮 | 说明 |
|------|------|------|
| 启用/禁用 | 「启用」或「禁用」 | 切换用户的启用状态 |
| 解锁 | 「解锁」 | 仅锁定用户显示，解除登录锁定 |
| 审核通过 | 「启用」 | 将待审核用户设为启用状态 |

**用户状态说明：**

| 状态 | 标签颜色 | 说明 |
|------|----------|------|
| 启用 | 绿色 | 正常使用 |
| 禁用 | 红色 | 无法登录 |
| 待审核 | 橙色 | 新注册用户，需管理员审核 |
| 待验证邮箱 | - | 注册后未完成邮箱验证 |

### 4.5 分配角色

1. 点击用户行的「分配角色」按钮
2. 在弹窗中勾选需要分配的角色（可多选）
3. 点击「确定」保存

### 4.6 重置密码

1. 点击用户行的「重置密码」按钮
2. 确认提示后，密码将被重置为 `123456`
3. 通知用户使用新密码登录后自行修改

### 4.7 删除用户

点击「删除」按钮并确认。删除为逻辑删除，数据不会物理清除。

## 五、角色管理

路径：系统管理 → 角色管理 `/system/role`

权限要求：需要「角色管理」菜单权限

### 5.1 预设角色

| 角色编码 | 角色名称 | 数据范围 | 说明 |
|----------|----------|----------|------|
| ADMIN | 系统管理员 | 全部数据 | 拥有所有权限 |
| ENGINEER | 运维工程师 | 本部门数据 | 仪表盘、用户管理、日志 |
| ANALYST | 数据分析师 | 本部门数据 | 仪表盘、日志 |
| VIEWER | 普通查看用户 | 仅本人数据 | 仅仪表盘 |

### 5.2 查询角色

支持按角色名称和状态筛选。

### 5.3 新增/编辑角色

| 字段 | 必填 | 说明 |
|------|------|------|
| 角色编码 | 是 | 唯一标识，如 ADMIN、ENGINEER |
| 角色名称 | 是 | 显示名称 |
| 描述 | 否 | 角色说明 |
| 状态 | 否 | 启用/禁用 |
| 排序 | 否 | 显示顺序 |

### 5.4 分配权限

1. 点击角色行的「分配权限」按钮
2. 在权限树中勾选需要的权限节点
3. 勾选父节点会自动选中所有子节点
4. 点击「确定」保存

### 5.5 配置数据范围

1. 点击角色行的「数据范围」按钮
2. 在下拉框中选择数据范围类型：

| 类型 | 标签颜色 | 说明 |
|------|----------|------|
| 全部数据 | 蓝色 | 可查看系统中所有数据，不受部门限制 |
| 本部门数据 | 绿色 | 仅可查看本人所在部门的数据 |
| 本部门及下级数据 | 青色 | 可查看本人所在部门及其所有下级部门的数据 |
| 仅本人数据 | 橙色 | 仅可查看本人创建/关联的数据 |

3. 弹窗底部会显示所选类型的详细说明
4. 点击「确定」保存

### 5.6 删除角色

点击「删除」按钮并确认。已分配给用户的角色建议先解除关联再删除。

## 六、权限管理

路径：系统管理 → 权限管理 `/system/permission`

权限要求：需要「权限管理」菜单权限

### 6.1 权限树结构

权限以树形表格展示，支持三种类型：

| 类型 | 标签 | 说明 |
|------|------|------|
| MENU | 菜单 | 左侧导航菜单项 |
| BUTTON | 按钮 | 页面内的操作按钮 |
| API | 接口 | 后端 API 接口权限 |

### 6.2 预设权限树

```
├── 数据仪表盘 (dashboard)                    [菜单]
└── 系统管理 (system)                          [菜单]
    ├── 用户管理 (system:user)                 [菜单]
    │   ├── 新增用户 (system:user:add)         [按钮]
    │   ├── 编辑用户 (system:user:edit)        [按钮]
    │   ├── 删除用户 (system:user:delete)      [按钮]
    │   └── 分配角色 (system:user:assign)      [按钮]
    ├── 角色管理 (system:role)                 [菜单]
    │   ├── 新增角色 (system:role:add)         [按钮]
    │   ├── 编辑角色 (system:role:edit)        [按钮]
    │   ├── 删除角色 (system:role:delete)      [按钮]
    │   └── 分配权限 (system:role:assign)      [按钮]
    ├── 权限管理 (system:permission)           [菜单]
    │   ├── 新增权限 (system:perm:add)         [按钮]
    │   ├── 编辑权限 (system:perm:edit)        [按钮]
    │   └── 删除权限 (system:perm:delete)      [按钮]
    └── 操作日志 (system:log)                  [菜单]
```

### 6.3 新增/编辑权限

| 字段 | 必填 | 说明 |
|------|------|------|
| 上级权限 | 否 | 选择父级节点，顶级为空 |
| 权限编码 | 是 | 唯一标识，如 system:user:add |
| 权限名称 | 是 | 显示名称 |
| 权限类型 | 是 | MENU / BUTTON / API |
| 路由路径 | 否 | 菜单类型需要，如 /system/user |
| 图标 | 否 | 菜单图标，如 icon-user |
| 组件路径 | 否 | 前端组件路径 |
| 排序 | 否 | 同级排序 |
| 状态 | 否 | 启用/禁用 |

### 6.4 删除权限

存在子权限时需先删除子权限，否则无法删除。

## 七、操作日志

路径：系统管理 → 操作日志 `/system/log`

权限要求：需要「操作日志」菜单权限

### 7.1 操作日志

记录用户在系统中的关键操作行为。

| 字段 | 说明 |
|------|------|
| 操作用户 | 执行操作的用户名 |
| 操作模块 | 如：用户管理、角色管理 |
| 操作类型 | 如：新增用户、修改角色 |
| 请求方法 | HTTP 方法 |
| 请求地址 | API 路径 |
| IP 地址 | 操作者 IP |
| 耗时 | 请求处理时间（毫秒） |
| 状态 | 成功/失败 |
| 操作时间 | 时间戳 |

支持按用户名和模块筛选。

### 7.2 登录日志

记录所有登录和登出行为。

| 字段 | 说明 |
|------|------|
| 用户名 | 登录用户 |
| IP 地址 | 登录来源 IP |
| 浏览器信息 | User-Agent |
| 登录类型 | 登录/登出 |
| 状态 | 成功/失败 |
| 消息 | 成功或失败原因 |
| 时间 | 时间戳 |

支持按用户名和状态筛选。

## 九、安全说明

### 9.1 密码安全

- 密码使用 BCrypt 算法加密存储，不可逆
- 密码最少 6 位，建议使用字母+数字+特殊字符组合
- 连续 5 次输错密码将自动锁定账户

### 9.2 会话安全

- 系统使用 JWT Token 认证，Token 有效期为 24 小时
- Token 过期后需重新登录
- 退出登录会清除本地 Token

### 9.3 输入安全

- 系统对所有用户输入进行 XSS 防护处理
- 包含恶意脚本的输入会被自动过滤或拒绝
- 所有写操作受 CSRF Token 保护

### 9.4 操作审计

- 所有关键操作（增删改）自动记录操作日志
- 所有登录行为自动记录登录日志
- 日志不可篡改，供安全审计使用

## 十、常见问题

### Q1: 注册后无法登录？

检查以下步骤：
1. 是否已完成邮箱验证（查收验证邮件并点击链接）
2. 邮箱验证后，是否已被管理员审核通过
3. 联系管理员确认账户状态

### Q2: 忘记密码怎么办？

联系系统管理员进行密码重置，重置后密码为 `123456`，登录后请立即修改。

### Q3: 账户被锁定怎么办？

连续输错密码 5 次会触发锁定。联系管理员在用户管理中点击「解锁」按钮。

### Q4: 看不到某些菜单或功能？

菜单和功能按钮受角色权限控制。联系管理员确认您的角色是否分配了对应权限。

### Q5: 验证邮件没收到？

1. 检查垃圾邮件/垃圾箱
2. 确认注册时填写的邮箱地址正确
3. 在注册成功提示中点击「未收到邮件？重新发送」
4. 验证链接有效期 30 分钟，过期需重新发送

### Q6: 页面显示异常？

1. 清除浏览器缓存后刷新
2. 确认使用推荐的浏览器版本
3. 检查网络连接是否正常
